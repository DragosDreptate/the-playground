Lecture rapide. V1 (pure cover) maximise l'impact visuel mais repose à 100% sur la cover et l'og:title pour transmettre l'info — si la cover est peu marquante, le résultat est anonyme. V6 (cover + titre + date) garantit le contexte même quand la cover est ambiguë, au prix d'un peu d'overload sur les previews très réduites (Slack icon).

Mon analyse rapide

  • V4 (cover + branding + date pill) est l'équilibre le plus sain pour démarrer : cover dominante, branding discret, info-clé (la date) saillante. Préserve l'esprit Luma + signe la plateforme.
  • V6 (cover + titre + date overlay bas) est plus dense mais fonctionne dans tous les contextes — utile quand un Host partage une URL sans message accompagnant.
  • V1 (pure Luma) est tentant mais risqué : le branding disparaît complètement et un Circle ressemble visuellement à un event si on ne distingue pas via l'overlay.
  • V8 (bandeau bas plein) est le plus lisible mais consomme 20% de la cover — moins beau mais robuste si certains organisateurs ont des covers difficiles (texte trop sombre, etc.).
