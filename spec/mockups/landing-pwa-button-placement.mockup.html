Hiérarchie dans le Hero
1. Commencer gratuitement (primary, gradient)
2. Télécharger l'app ← ici
Respect de la règle design system : un seul bouton default par section. Le bouton PWA est outline — action secondaire.
Logique d'affichage
Affiché si : · PWA non encore installée · beforeinstallprompt disponible · OU iOS Safari (prompt manuel)
Masqué si : · App déjà installée · Navigateur non compatible
Contexte de placement — Pourquoi ici ?
Le bouton "Télécharger l'app" est placé dans le Hero parce que c'est la seule section visible sans scroll sur mobile. Les visiteurs qui arrivent via un lien WhatsApp ou Instagram voient immédiatement les deux options : s'inscrire sur le web, ou installer l'app.

La hiérarchie visuelle est respectée : le gradient rose/violet du CTA principal capte l'oeil en premier, le bouton outline secondaire est accessible sans effort mais ne concurrence pas l'action principale.

Le bouton n'apparaît pas si l'app est déjà installée (détection via window.matchMedia('(display-mode: standalone)')), ce qui élimine le bruit visuel pour les utilisateurs existants.
