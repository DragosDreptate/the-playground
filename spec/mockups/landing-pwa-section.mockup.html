Note PWA
La note sous les badges précise qu'il s'agit d'une PWA — aucun vrai téléchargement App Store/Play Store n'est requis. C'est un point de réassurance pour les utilisateurs méfiants.
Pourquoi ajouter la section PWA après le CTA final plutôt que de le remplacer ?
La section "Prêt à lancer votre Communauté ?" est conservée : c'est un CTA de conversion directe vers l'inscription, qui reste pertinent en bas de page pour l'utilisateur qui a scrollé jusqu'au bout.

La section PWA est ajoutée juste après comme deuxième couche de conversion : l'utilisateur qui a scrollé encore plus loin est très engagé — c'est le moment idéal pour lui proposer d'installer l'app. Les deux sections ciblent des intentions différentes (créer un compte vs. installer l'app).

L'alternance de fond est préservée : le CTA final utilise bg-muted, la section PWA utilise bg-background (blanc) pour maintenir le rythme visuel muted / white.

La modale iOS est nécessaire car Safari sur iPhone n'a pas de popup native beforeinstallprompt — les 3 étapes manuelles doivent être expliquées. Sur Android, la popup native gère tout directement.
