03 — Notes d'implémentation
Détection de plateforme et logique du bouton
  • iOS/Safari : navigator.standalone absent + userAgent contient iPhone/iPad → afficher la bottom sheet avec les 3 étapes manuelles.
  • Android/Chrome : Écouter window.beforeinstallprompt → stocker l'événement → au clic sur le bouton, appeler deferredPrompt.prompt(). Si l'événement n'est pas disponible (déjà installé ou navigateur non supporté) → masquer le bouton ou afficher un message "Déjà installée".
  • Desktop Chrome/Edge : Même logique que Android — l'API est supportée. Le prompt natif s'affiche en haut à droite dans Chrome desktop.
  • Déjà installé : Si window.matchMedia('(display-mode: standalone)').matches → masquer le bouton complètement.
  • Autres : Bouton masqué par défaut si ni iOS ni PWA prompt disponible.
Positionnement bouton : Le bouton "Télécharger l'app" n'est affiché que si l'installation PWA est pertinente pour le contexte détecté. Il reste en outline (secondaire) — le CTA principal "Commencer gratuitement" reste toujours visible et dominant.
